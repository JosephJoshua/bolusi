### 2.5 Server tenancy tests (RLS)

**Under D16 this suite runs on real PostgreSQL 16 in a container, over the real `pg` driver â€” not PGlite.** The owner-bypass trap below is the reason it was never safe on a substitute, and it does **not** go away by switching engines: **testcontainers' default `postgres` user is a SUPERUSER, and superusers bypass RLS even under `FORCE ROW LEVEL SECURITY`.** Swapping PGlite for a container while still connecting as the default user would swap one vacuous RLS lane for another and call it progress (D16 says this in as many words). What makes the lane non-vacuous is the ROLE, not the engine. Therefore, normative:

1. 2. A catalog assertion runs in CI: every table carrying a `tenant_id` column has `pg_class.relrowsecurity = true` and at least one policy â€” a new tenant table without RLS fails the build.
3. ### 3.4 The convergence oracle (defined once; every convergence assertion uses it)

`digest(db, module)` â€” engine-neutral, byte-exact:

1. Tables = the module's projection tables per its manifest, ascending byte order of table name. **Excluded:** the op log, watermark tables (`applied_server_seq`/`applied_local_seq`), all client bookkeeping (`syncStatus`, `syncedAt`, â€¦), `SyncState`, `quarantined_ops`.
2. For each table, `SELECT` the manifest-declared columns in declaration order â€” all rows, **no SQL ORDER BY** (collation differs across engines; sorting happens in JS).
3. Normalize each scalar:

   | Stored value | Normalized |
   | ------------ | ---------- |
   | NULL | `null` |
   | integer (incl. pg `int8` returned as string) | JSON integer; if magnitude > 2^53 âˆ’ 1 â†’ decimal string |
   | boolean-declared column (pg `true/false`, SQLite `0/1`) | `1` / `0` |
   | text | JSON string |
   | blob / `bytea` | `"0x"` + lowercase hex |
   | any float / non-integer numeric | **oracle ERROR** â€” floats are banned from projections (`05 Â§3`); the oracle enforces it |

4. Each row â†’ `JCS([tableName, v1, â€¦, vn])` (RFC 8785 via the same shared implementation ops use, `05 Â§3`).
5. Sort all row-lines of the whole module ascending by UTF-8 byte order; `digest = SHA-256( join(lines, "\n") + "\n" )`.

**Convergence** = digests byte-equal. The **canonical-fold reference** for an op set: fresh DB via the shim, insert nothing, feed all ops to the production projection engine strictly in canonical order `(timestamp ASC, deviceId ASC, seq ASC)` (`05 Â§4`), then `digest()`. Every convergence scenario asserts: each device's digest == every other device's == the server's == the canonical-fold reference.
