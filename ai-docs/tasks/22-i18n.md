Gate tests (each gate proven by a failing fixture, run as unit tests against fixture catalogs):

- **Error-code coverage** — fixture catalog missing `core.errors.ROLE_IN_USE` → gate fails naming the code; fixture missing `core.rejection.CHAIN_HALTED` → fails; full seed (all 12 DomainError codes of 04 §5.3 + `IDEMPOTENCY_CONFLICT` + `RATE_LIMITED` + `UNEXPECTED` + all 8 rejection codes) → passes.
- **Key grammar / collision** — fixtures that must fail: 2-segment key `core.save`; non-camelCase segment `core.Action.save`; same key defined in two catalogs; module id equal to a reserved namespace. Fixtures that must pass: derived keys `core.rejection.CHAIN_BROKEN`, `permission.notes.create.name`, `role.main_owner.name` (the three §3.1 exceptions).
- **ICU subset** — fixture value with `{x, number}` (forbidden feature) → fails; unparseable ICU → fails; `plural` block inside an `id` value → fails (id uses plain `{count}` per §3.2); ASCII `'` swallowing text in an `en` value is caught (parse or explicit check).
- **Parity** — fixture where `en` lacks a key present in `id` → fails; `zh` absence → does not fail (exempt).
- **Extraction** — fixture source using `t('auth.pin.nonexistent')` → fails; unused catalog key → warning only.

Formatter + runtime tests:

- Golden vectors exactly per 07 §5.4 table: `formatMoney(250000) === 'Rp 250.000'` (both locales), `Rp 0`, `Rp 1.500.500`, `-Rp 50.000`; assert output contains **no** U+00A0 (NBSP normalized); `formatDate` → `14/07/2026` both locales; `formatTime` → `14.30` (id) / `14:30` (en); `formatNumber(1234567)` → `1.234.567` / `1,234,567`.
- `formatRelative` boundary tests select `core.time.justNow` (<60 s) / `minutesAgo` / `hoursAgo` / `daysAgo`; `formatDuration` selects `durationSeconds`/`durationMinutes`.
